SECTION 6: METHOD SETUP (1–3 pages)
-----------------------------------------------------------------------------

Core idea:
  SlideQA is a multimodal QA benchmark over lecture slides requiring
  visual understanding that cannot be solved by text alone. We evaluate
  four systems across a retrieval-augmented generation pipeline:
  closed-book → text-only BM25 retrieval → ColPali visual retrieval → oracle VLM.
  The main engineering contribution is the ColPali RAG pipeline.

ColPali RAG Pipeline (the main method):
  1. Offline index build (slideqa/src/build_index.py):
     - Model: vidore/colpali-v1.2 (patch-based visual encoder, PaliGemma backbone)
     - Each slide PNG is embedded into a (1031, 128) patch embedding matrix
     - Stored as numpy arrays: slideqa/data/colpali_index/{course}/embeddings.npy
     - Slide IDs (lecture/slide path strings): slide_ids.json (same dir)
     - Hardware: Apple M-series GPU (MPS), batch_size=4, ~4.5 hrs per course
     - Command: python -u slideqa/src/build_index.py --course cs288

  2. Query-time retrieval (slideqa/src/baselines/colpali_rag.py):
     - Query is embedded with the same ColPali model (processor handles text)
     - ColBERT-style MaxSim scoring:
         score(slide) = Σ_p max_q(slide_patch_p · query_emb_q)
       where the sum is over all 1031 slide patches and max over query tokens
     - Top-5 slides retrieved per question; top-1 slide image sent to GPT-4o
     - GPT-4o (via OpenRouter) generates the final answer from the image

  3. Recall@k logging (slideqa/src/recall_at_k.py):
     - Checks whether the gold evidence slide appears in top-k retrieved slides
     - k ∈ {1, 2, 3, 5}

Theoretical framing:
  ColBERT-style late interaction allows fine-grained patch-level matching
  between query tokens and visual regions, unlike single-vector retrieval
  (e.g. CLIP) which compresses all visual information into one embedding.
  This matters for slides because the answer is often in a specific region
  (a cell in a table, one point in a bullet list, one bar in a chart).

Other baselines (for comparison):
  - closed_book (slideqa/src/baselines/closed_book.py): no context, question only
  - text_only (slideqa/src/baselines/text_only.py): BM25 over slide OCR text
  - zero_shot_vlm (slideqa/src/baselines/zero_shot_vlm.py): gold slide image + GPT-4o
    (oracle upper bound — "what if retrieval were perfect?")

Evaluation metrics:
  - Exact Match (EM): strict string match after normalization
  - Token F1 (SQuAD-style): overlap of predicted and gold answer tokens
  - LM-as-judge: GPT-4o scores predicted answer vs gold on 1–5 scale
    (justified because EM/F1 underestimate quality on open-ended answers)
  - Recall@k: fraction of questions where gold slide appears in top-k
  - Script: slideqa/src/evaluate.py (EM + F1), slideqa/src/run_judge.py (judge)

SECTION 8: ANALYSIS (within 1 page)
-----------------------------------------------------------------------------

Ablation study (retrieval ablation):
  The four baselines form a natural ablation ladder:
    closed_book  →  text_only (BM25)  →  colpali_rag  →  oracle_vlm
  Each step adds one capability: (1) text retrieval, (2) visual retrieval,
  (3) perfect retrieval. F1 increases monotonically at each step.
  The colpali_rag → oracle gap (ΔF1 ≈ 0.08) is the retrieval error gap,
  directly predicted by 1 - Recall@5 ≈ 0.20.

Recall@k curve as ablation:
  Recall@1 vs Recall@5 shows the headroom from re-ranking or
  sending top-k images to the VLM instead of just top-1.
  E.g., for image_diagram on CS 288: R@1=0.367 but R@5=0.717 — retrieving
  the right slide is achievable in top-5, but top-1 precision is still low.

Error analysis (findings from per-category breakdown):

  1. image_diagram is the hardest retrieval category (R@1 ≈ 0.35 avg).
     Complex diagrams with similar visual structure (flowcharts, system
     diagrams) produce similar ColPali patch embeddings, causing false-positives.
     This also produces the lowest ColPali F1 despite the oracle gap being large.

  2. chart_graph and table have the strongest retrieval (R@1 ≈ 0.61–0.76).
     Charts have highly distinctive visual fingerprints (bar heights, line shapes)
     that ColPali captures well. Tables have strong structural regularity.

  3. text_only BM25 retrieval is hurt by slide OCR noise (missing text from
     images, equations, etc.) and results in lower judge scores than closed_book
     on CS 288 and CS 224N — retrieved-but-irrelevant context actively
     misleads GPT-4o into confidently wrong answers.

  4. CS 601 (robotics) has the weakest ColPali retrieval overall (R@5=0.707).
     Robotics slides have visually repetitive content (robot arm poses,
     coordinate frames, motion diagrams) that creates retrieval ambiguity.

Qualitative examples to include (pick from judge_details.json):
  - A chart_graph question where text_only returns wrong text, ColPali finds
    the right bar chart, VLM reads the value correctly.
  - An image_diagram question where ColPali retrieves a similar-but-wrong

  - A text_only question where all baselines succeed (simple factual recall).
  Relevant files for finding examples:

SECTION 9: LIMITATIONS AND BROADER IMPACT (1–2 paragraphs, optional)
-----------------------------------------------------------------------------

Limitations:
  - Scale: 150 QA pairs per course is small; single annotator introduces
    bias. The automated generation + filtering pipeline is available to scale
    (slideqa/src/generate_qa.py and slideqa/src/annotate.py).
  - Retrieval upper bound: top-1 image to VLM. Sending top-3 would likely
    improve ColPali RAG F1 substantially (R@3 is 0.75–0.81 vs R@1 at 0.43–0.50).
  - ColPali index cost: ~4.5 hrs per course on Apple M-series; not practical
    in a real deployment without GPU hardware.
  - Courses are all English, university-level CS. Domain transfer unknown.
  - Evidence slides were identified by the question author — "ground truth"
    retrieval may itself be incomplete for multi-slide questions.

Broader impact:
  - SlideQA could accelerate studying and accessibility in education.
  - Risk of over-reliance: incorrect VLM answers may mislead students
    who don't cross-check sources.
  - Slide content is copyright of the original instructors; dataset use
    should be restricted to research.

SECTION 10: CONCLUSION (1 paragraph)
-----------------------------------------------------------------------------

Key points to hit:
  - We introduced SlideQA, a multimodal QA benchmark across 450 slide-based
    questions from 3 university CS courses spanning 5 visual categories.
  - We showed that visual content is essential: the text-only baseline
    scores well below the oracle VLM (F1 gap of 0.23–0.29), and the gap
    is largest for image_diagram and chart_graph questions.
  - Our ColPali RAG pipeline closes ~60% of this gap (F1 ≈ 0.30 vs oracle
    ≈ 0.38) without any oracle slide access, using patch-level visual retrieval.
  - LM-as-judge scores show an even cleaner story: ColPali RAG ≈ 3.6/5
    vs text-only ≈ 1.8/5, a 2× improvement.
  - Future work: top-k image re-ranking, fine-tuned VLM answering,
    scaling to more courses and question types.
