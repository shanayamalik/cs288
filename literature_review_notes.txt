CORE REFERENCES
========================================================================

1. SlideVQA  [MOST IMPORTANT — closest existing benchmark]
   - Authors: Tanaka, Nishida, Nishida, Hasegawa, Saito, Saito
   - Venue: AAAI 2023  |  arXiv: 2301.04883
   - Size: 14.5k questions, 52k+ slide images, 2.6k+ slide decks
   - Domain: General-domain slide decks (business, presentations)
   - Multi-page: Yes (questions can span multiple slides)
   - Question types: Single-hop, multi-hop, numerical reasoning
   - Summary: First large-scale multi-image document VQA dataset on slides.
     Proposes an end-to-end model treating evidence selection + QA as a
     unified sequence-to-sequence task. Still has a large gap vs human perf.
   - How SlideQA differs:
     (a) SlideQA is education-specific (NLP courses) — slides contain
         technical content (math, ML diagrams, pseudocode) unlike the
         general-domain presentations in SlideVQA.
     (b) SlideQA has a 5-category modality taxonomy enabling fine-grained
         evaluation per visual-reasoning type. SlideVQA only distinguishes
         single-hop / multi-hop / numerical.
     (c) SlideQA jointly evaluates retrieval + QA via a RAG pipeline.
         SlideVQA assumes the candidate slide deck is given.

2. DocVQA  [foundational document VQA benchmark — everyone cites this]
   - Authors: Mathew, Karatzas, Jawahar
   - Venue: WACV 2021  |  arXiv: 2007.00398
   - Size: 50k questions, 12k+ document images
   - Domain: Industry documents (invoices, reports, forms)
   - Multi-page: No (single page only)
   - Summary: Foundational benchmark for VQA on document images. Large gap
     between models and humans (94.36% acc), especially on questions requiring
     structural understanding of documents.
   - How SlideQA differs: DocVQA is single-page industry documents. SlideQA
     targets multi-slide educational content with richer visual elements
     (diagrams, math expressions, charts) and includes a retrieval stage.

3. ChartQA  [covers our chart_graph question category]
   - Authors: Masry, Long, Tan, Joty, Hoque
   - Venue: ACL Findings 2022  |  arXiv: 2203.10244
   - Size: 9.6k human-written + 23.1k generated questions (32.7k total)
   - Domain: Charts only
   - Summary: Focuses on complex visual and logical reasoning over charts.
     Proposes transformer models combining visual features with chart data
     tables. Includes both human-authored and machine-generated questions.
   - How SlideQA differs: Chart-only and not embedded in slide context.
     SlideQA's "chart_graph" category covers this as one of five question
     types within the broader lecture slide setting.

4. ColPali  [core to our retrieval pipeline]
   - Authors: Faysse, Sibille, Wu, Omrani, Viaud, Hudelot, Colombo
   - Venue: ICLR 2025  |  arXiv: 2407.01449
   - Summary: Vision Language Model for document retrieval that directly
     embeds page images — no OCR or text extraction needed. Uses late
     interaction matching (ColBERT-style) on vision token embeddings.
     Introduces the ViDoRe benchmark. Outperforms text-based retrieval
     pipelines while being simpler and end-to-end trainable.
   - Relevance: This is the retrieval approach we're using to embed slides
     in our multimodal RAG pipeline.

5. RAG  [foundational paper for retrieval-augmented generation]
   - Authors: Lewis, Perez, Piktus, Petroni, Karpukhin, et al.
   - Venue: NeurIPS 2020  |  arXiv: 2005.11401
   - Summary: Combines a pre-trained retriever (DPR) with a seq2seq
     generator. Shows that retrieval-augmented models outperform purely
     parametric models on knowledge-intensive NLP tasks.
   - Relevance: Cite for the general retrieval-augmented pipeline design.
     Our system extends RAG to the multimodal (vision) setting.

========================================================================
KEY DIFFERENTIATION ARGUMENTS (for the closing paragraph)
========================================================================

1. No existing benchmark focuses on EDUCATIONAL LECTURE CONTENT.
   SlideVQA is closest but uses general-domain slide decks. SlideQA targets
   university NLP courses with domain-specific technical content.

2. No existing benchmark provides a MODALITY-GROUNDED CATEGORY TAXONOMY.
   Most benchmarks have free-form questions without categorizing which type
   of visual reasoning is required. SlideQA's 5 categories enable
   fine-grained evaluation of where models succeed or fail.

3. No existing benchmark jointly evaluates RETRIEVAL + QA over slides.
   Existing benchmarks assume the relevant document is given. SlideQA
   includes a retrieval stage, matching real-world RAG scenarios.
